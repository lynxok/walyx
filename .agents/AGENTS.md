# Reglas y Guías para Subagentes (Venta de Viandas - Cocina)

Este archivo define las reglas de estilo, restricciones de comportamiento e instrucciones generales para todos los agentes y subagentes que colaboren en el desarrollo del proyecto **Venta de Viandas (Cocina)**.

## 1. Contexto del Proyecto
El proyecto tiene como objetivo desarrollar un sistema para la gestión y venta de viandas (cocina).
- **Frontend**: Diseños premium, limpios, responsivos, utilizando vanilla CSS o TailwindCSS (según se solicite), priorizando una experiencia de usuario fluida y estética moderna.
- **Backend/Base de Datos**: Integración con Supabase, APIs locales, o base de datos relacional.

## 2. Pautas de Comportamiento de los Subagentes
- **Investigación Eficiente**: Antes de realizar cualquier modificación, el subagente de investigación (`research`) o cualquier subagente especializado debe inspeccionar el código existente y sus dependencias.
- **Sin Cambios Inesperados**: Los subagentes no deben realizar modificaciones fuera del alcance asignado sin autorización del agente principal.
- **Comunicación Concisa**: Toda comunicación entre subagentes debe ser clara, documentando los hallazgos y avances específicos.
- **Mantenimiento de Código**: Conservar comentarios y documentación preexistente que no esté relacionada directamente con los cambios solicitados.

## 3. Estructura de Trabajo
- Las tareas complejas se dividen en subtareas asignadas a subagentes específicos.
- Cada subagente debe reportar al finalizar su tarea con un resumen detallado y los archivos modificados/creados.
